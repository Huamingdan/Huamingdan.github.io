1.2函数的原型

​ 1、对象有一个_ proto_，他指向构造函数的原型

​ 2、函数有一个prototype，他表示函数自己的原型信息

​ 3、在函数的原型中有三个部分，分别是公共的属性和方法、constructor、_ proto _

其实constructor指向当前函数本身，_ proto _指向父亲的原型信息

​ 4、其实每个函数都是通过new Function()创建出来的，所以函数本身也是对象

所以函数._ proto _ == Function.prototype

​ 5、Function.prototype._ proto _ == Object.prototype

​ 6.在Function.prototype中有call bind apply等方法

1.3 call bind apply的使用

call和apply方法的作用：修改被调用函数中的this指向，直接调用当前函数

call方法参数:

​ f1.call(入参谁this就是谁,原函数中的参数1,原函数中的参数2);

​ 注意点：call方法的第一个参数如果是null或者undefined，那么原函数中的this仍然是window

call和apply方法的区别在于 apply方法第二个参数是一个数组

f1.call(obj,100,200);

f1.apply(obj,[1000,2000]);

bind方法会修改指定函数中的this指向，会返回一个新函数

var ff = f2.bind(obj); //参数obj就是要修改的this

ff(11,22);

1.6伪数组

什么是伪数组?

伪数组具备数组的部分特性，比如具备length，具备下标，但是不可以调用数组的方法

伪数组问题：一旦删除伪数组中的元素之后，伪数组length属性不会同步更新

伪数组的原理:其实是通过对象来模拟数组

var obj = {

          0:"张三",

          1:"李四",

​ 2:”王五”,

​ length:3

​ }

​ console.log(obj[0]);

console.log(obj[1]);

console.log(obj[2]);

delete obj[1];

​ console.log(obj);

我们可以使用call apply等方法让伪数组调用真正数组的方法
Array.prototype.forEach.call(arguments,function(e){

          console.log(e);

})

Array.prototype.push.call(arguments,”xxx”);

​ console.log(arguments);

1.10闭包

闭包可以理解为 在函数中嵌套函数

闭包内部函数可以访问外部函数的变量

闭包缓存数据,延长变量的作用域链

​ 闭包的内存问题：一个函数执行开始时候，会给其中定义的变量划分内存空间保存，以备后面的语句使用，等函数执行完毕返回了这些变量被认为无用的了，对应的空间也就收回了，下次执行此函数的时候，所有变量回到初始状态，重新赋值使用，但是这个函数内部又嵌套一个函数，而这个函数有可能被外部调用，并且这个函数又使用了外部函数某些变量这种内存回收机制就出现问题。如果外部函数返回后，又直接调用了内部函数，那么内部函数就无法读取到它所需要的外部函数中变量的值，所以js解释器在遇到函数定义的时候会自动把函数和他可能使用到的变量一起存起来，也就构建了一个闭包，这些变量将不会被内存回收，只用当内部函数不可能被调用以后，才会销毁这个闭包，而没有任何一个闭包引用的变量才会被下次内存回收启动时回收
